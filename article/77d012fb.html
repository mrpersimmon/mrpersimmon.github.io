Collection 接口和常用方法

Collection 接口实现类的特点

public interface Collection<E> extends Iterable<E>

  1. Collection 实现子类可以存放多个元素，每个元素可以是 Object；
  2. 有些 Collection 的实现类，可以存放重复的元素，有些不可以；
  3. 有些 Collection 的实现类，有些是有序的（List），有些不是有序（Set）；
  4. Collection 接口没有直接的实现子类，是通过它的子接口 Set 和 List 来实现的。

下面，以 List 接口的实现类 ArrayList 为例，演示常用方法的使用。

方法名 作用
add 添加单个元素
remove 删除指定元素
contains 判断元素是否存在
size 获取集合中元素的个数
isEmpty 判断集合是否为空
clear 清空集合
addAll 添加多个元素
containsAll 判断多个元素是否都在
removeAll 删除多个元素

使用 Iterator（迭代器）遍历 Collection 接口元素

Iterator<E> 接口的基本介绍

  1. Iterator 对象称为迭代器，主要用于遍历 Collection 集合中的元素；
  2. 所有实现了 Collection 接口的集合类都有一个 iterator() 方法，用以返回一个实现了 Iterator 接口的对象，即可以返回一个迭代器。
  3. Iterator 仅用于遍历集合，Iterator 本身并不存放对象。

迭代器的执行原理

具体流程如下：

  1. 得到一个集合的迭代器；
  2. 使用 it.hasNext() 循环判断集合中是否还有下一元素；
  3. 如果有元素，使用 it.next() 方法下移指针并返回下移后的集合元素。
